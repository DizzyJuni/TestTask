## О проекте

REST-сервис для управления кошельками с поддержкой конкурентных операций (1000+ RPS).

**Функционал:**

- Пополнение баланса (`DEPOSIT`)
- Снятие средств (`WITHDRAW`)
- Получение текущего баланса

**Особенности:**

- Пессимистическая блокировка для работы в конкурентной среде
- Контейнеризация (Docker + docker-compose)
- Миграции базы данных (Liquibase)
- Документация API (Swagger)
- Обработка ошибок и валидация запросов

## Запуск проекта

### Требования к системе

- Установленный Docker (https://www.docker.com/products/docker-desktop/)
- Docker Compose (входит в Docker Desktop)

### Команда для запуска

```bash
docker-compose up --build
```

После запуска приложение будет доступно по адресу: http://localhost:8080

## Остановка

```bash
docker-compose down
```

## Полная очистка (с удалением данных БД)

```bash
docker-compose down -v
```

# Проверка работы API

## Документация Swagger

После запуска откройте в браузере:
👉 http://localhost:8080/swagger-ui.html

Там можно увидеть все эндпоинты и отправить тестовые запросы.

## Конкурентность (1000+ RPS)

Для обеспечения работы при высоких нагрузках используется:

- Пессимистическая блокировка (@Lock(LockModeType.PESSIMISTIC_WRITE))
- SELECT FOR UPDATE на уровне базы данных
- Уровень изоляции транзакций READ_COMMITTED

## Результаты нагрузочного тестирования

Запуск 200 одновременных запросов на пополнение одного кошелька:

- ✅ Все запросы успешно обработаны (ошибок 50X нет)
- ✅ Итоговый баланс совпал с ожидаемым

## Настройка параметров без пересборки

Все параметры вынесены в переменные окружения. Для изменения достаточно создать файл .env в корне проекта:

```env
# База данных
DB_NAME=walletdb
DB_USER=postgres
DB_PASSWORD=mysecretpassword
DB_PORT=5432

# Приложение
APP_PORT=8080
```

Или передать переменные при запуске:

```bash
DB_PASSWORD=newpass APP_PORT=9090 docker-compose up
```
